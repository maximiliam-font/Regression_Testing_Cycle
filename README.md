####  Requirements Analysis and Planning

- Dependencies between fields and transport modes.

---

#### Regression Test Execution
During test execution:
- Followed all steps detailed in the test case template (ID, title, precondition, step description, expected result).
- Assigned status to each test case: Passed, Failed, or Skipped.
- Compared actual results against expected results, identifying discrepancies representing application defects.
- Applied logical reasoning to anticipate issues not explicitly covered in the test cases.
<br/>
These actions validated the stability of the application after recent changes and ensured proper functionality of critical flows.

---

##  Bug Documentation
For each failure detected:
- Created detailed bug reports including ID, title, steps, expected result, and actual result.
- Classified each defect by severity: Blocker, Critical, Major, Minor, or Trivial.
- Ensured reports were clear, reproducible, and actionable, enabling the development team to address issues efficiently.
<br/>
The documentation contributes to complete traceability between test cases and detected defects.
